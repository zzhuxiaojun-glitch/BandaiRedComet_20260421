# 10. 使用指南 · 5 分钟上手

**适用场景**：你或朋友要用这个脚本抢万代南梦宫上海微商城的 PB 限定商品。
**前置假设**：工作区已 `git clone` 到本地、Python 3.11+ 已装、`.venv` 已建好。

---

## 10.0 两种使用模式

**根据你是谁选**：

| 角色 | 推荐模式 | 说明 |
|---|---|---|
| **朋友（技术小白）** | GUI 模式（§10.10） | 双击 exe / 点按钮，无需命令行 |
| **你自己 / 高级用户** | CLI 模式（§10.1 - §10.9） | 更轻、可脚本化 |

**当前状态（2026-04-23）**：
- CLI 模式：可用（生产级）
- GUI 模式：Phase 2 进行中，可在 WSLg / Windows 下运行源码；PyInstaller 打成 exe 在 **Phase 3** 完成

## 10.1 一次性环境准备（第一次运行前）

```bash
# WSL / Linux
cd <工作区>
python3 -m venv .venv
.venv/bin/pip install -r requirements.txt
```

如果要跑 GUI（可选）：
```bash
# Qt 后端 + WebEngine（必须）
.venv/bin/pip install pyqt6 qtpy PyQt6-WebEngine

# 中文 / 日文字体（WSL 用户必须，Windows 自带）
sudo apt install -y fonts-noto-cjk
```

验证装好：
```bash
PYTHONPATH=src .venv/bin/python -m bandai_sniper --help
# 应输出 ntp / check / run / gui 四个子命令
```

## 10.2 第二步：拿一个新鲜 CK

CK = `api-access-token`，一串 175 字符的 JWT。**每个抢购窗口前重抓一次最稳**（虽然 JWT 一般能活几天，但过期后请求全 401）。

### 方式 A：抓包（推荐，一次 2 分钟）

按 [`3_抓包指南.md`](./3_抓包指南.md) 用 mitmproxy `local:WeChat,WeChatAppEx` 模式抓。

流程：
1. Windows 管理员 PowerShell 跑 `tools/capture/start_capture.ps1`
2. 打开万代小程序，随便点两下触发一个接口
3. Ctrl+C 停止抓包 → 得到 HAR
4. 从 HAR 任意一条请求的 `api-access-token` header 复制那 175 字符

### 方式 B：小工具（无需装 mitmproxy）

朋友给的 `万代上号/上号软件/小程序抓Token.exe`，打开 → 进万代小程序 → 工具窗口显示 Token → 复制。

**把 CK 贴进 `config.yaml`** 或 `.env`（见下一步）。

---

## 10.3 第三步：配置 config.yaml

```bash
cp config.example.yaml config.yaml
nano config.yaml   # 或用 VSCode / Cursor 打开
```

必填 5 项：

```yaml
ck: "eyJ0eXAiOi...贴进来..."         # 10.2 拿到的 CK
snipe_time: "2026-05-10 16:00:00"     # 抢购开始时刻（本地时区）
target:
  spu_id: "6521"      # 从商品详情页或 HAR 拿
  sku_id: "9266"      # 同上
  address_id: "695133"  # 不填则自动用地址列表第一条
```

### 怎么拿 spu_id / sku_id / address_id？

**⚠️ 万代小程序禁止分享商品页**（分享菜单里 "当前页面不可分享" 是灰色的，"复制链接"给的是 `#小程序://...` 不透明短码，不包含 SPU ID）。所以**只能通过抓包**。

**方式 A（CLI）**：抓 HAR 后用小工具列商品
```bash
# 1. 按 docs 3 抓一次 HAR，抓包时进入你想抢的商品详情页
# 2. 跑 list_products_from_har.py，会输出表格
PYTHONPATH=src .venv/bin/python tools/verify/list_products_from_har.py \
  <path-to-har>
```
输出样例：
```
SPU ID   价格       库存   状态     开售时间                商品名
───────────────────────────────────────────────────────────────
6508     ¥250.0   1059   可售     2026-04-15 16:00:00    预售 PB 万代模型 MG 扎古加农
6521     ¥142.0   653    可售     2026-04-20 16:00:00    预售 PB 万代模型 HGUC 里歇尔
```

**方式 B（GUI，推荐）**：GUI 里点「📁 从 HAR 选商品」按钮
- 弹原生文件选择器 → 选你抓的 HAR
- 弹出商品列表，点一条就自动填 SPU ID + 拉 SKU
- 详见 §10.10

**address_id**：跑一次 `check` 子命令（或 GUI 点"先预检"），日志/界面会打印账户地址列表和 id。

## 10.4 第四步：跑预检（不抢，验证配置）

```bash
PYTHONPATH=src .venv/bin/python -m bandai_sniper check
```

**期望输出**：
```
INFO    预检 1/4: 服务器对时
INFO    预检 2/4: CK 有效性
INFO    ✅ CK 有效, memberId={796529}
INFO    预检 3/4: 商品 spuId=6521
INFO    ✅ 商品「预售 PB ...」 price=... stock=... saleStatus=0
INFO    预检 4/4: 地址
INFO    ✅ 地址 id=... ...
SUCCESS 预检通过，可以等时间
```

**预检挂了的常见原因**：

| 现象 | 可能原因 | 处理 |
|---|---|---|
| 全部请求返 `HTTPStatusError 432` | UA / Referer 被服务端拒 | 确认用了 `BandaiClient` 默认头（不是自己改过） |
| CK 相关：`code=302/703/307/1004` | CK 过期 | 重抓一次 CK |
| 商品查询 `saleStatus != 0` | 未到开售时间 | 这是正常的，预检 WARNING 不阻塞 |
| 地址 id 找不到 | `config.yaml` 填的 id 不在你账户下 | 用预检打印的 id 替换或留空自动选 |

## 10.5 第五步：NTP 对时（可选但推荐）

```bash
PYTHONPATH=src .venv/bin/python -m bandai_sniper ntp
```

输出 "本机时间偏差: **±XXXms**"。

- `|offset| < 100ms`：本机时钟准，直接抢
- `|offset| > 500ms`：本机时钟偏差大，脚本内置 NTP 校准会补回来，不影响抢购，但主机时钟不准可能影响日志时间戳
- NTP 全部失败：脚本会用本机时间 + 服务器 `sync_timestamp` 双重补正，一般不影响

## 10.6 第六步：抢！

```bash
PYTHONPATH=src .venv/bin/python -m bandai_sniper run
```

**时序**：
```
T-N秒（启动到开抢还剩 N 秒）
  |  precheck（同 check）
  |
T-60秒  进预热期：再对一次时、打开 HTTP/2 keep-alive
  |
T=0秒   触发 fire：3 个并发 worker 同时 confirmOrder + createOrder
  |
T+?     拿到 prepay_id → notify 推送订单号 → 退出
```

**看到这行就成了**：
```
SUCCESS 🎉 抢单成功 · 耗时 XXXms · order_id=... prepay_id=wx...
```

**15 分钟内**到微信小程序 → 我的订单 → 付款。

## 10.7 出问题怎么办

### 抢购时刻已到但脚本还没下单

1. 先看日志 `logs/run_*.log` —— 找 `ERROR` 和 `WARNING`
2. 看 `logs/requests_*.jsonl` —— 最后一条请求是什么状态

### 返 `code=` 某个业务码且一直重试

- 如果是"库存不足"、"限购"等终局错误，脚本会 fail-fast
- 如果是"请稍后再试"等可恢复错误，把那个 code 加到 `config.yaml` → `strategy.retryable_codes`，下次 run 会重试

### 真被万代风控

见 [`5_方案B_UI自动化规划.md`](./5_方案B_UI自动化规划.md) —— Plan B 兜底方案。

## 10.8 实战小贴士

1. **提前一天跑预检**：CK 没过期 + 商品信息对得上 = 当天可直接 run
2. **抢购前 5 分钟启动脚本**：`pre_warmup_seconds: 60` 给足预热时间
3. **抢到后马上付款**：15 分钟自动取消，别拖
4. **别连续多次抢同一个商品**：万代有"限购 N 件"规则，超了会在 createOrder 时 fail
5. **网络稳定比并发数重要**：`concurrency: 3` 已经够用，开到 10 不会更快反而可能触发限流

## 10.10 GUI 模式（给朋友 / 小白用户）

**现状**：GUI 源码可运行（Phase 1+2 已成），**exe 打包在 Phase 3**。目前朋友如果想用必须跑源码；Phase 3 之后会给 `万代抢购器.exe`。

### 启动 GUI

```bash
PYTHONPATH=src .venv/bin/python -m bandai_sniper gui
```

（加 `--debug` 会额外弹出 Web Inspector 开发者工具窗口，平时不加）

### GUI 内操作流程（5 步）

1. **① 账号**：把 CK 粘到输入框。勾选"记住 CK"下次打开自动预填（默认不存，敏感）
2. **② 商品**：SPU ID 两种填法：
   - **点 "📁 从 HAR 选商品"**（推荐）→ 选你抓的 HAR 文件 → 弹出商品列表，点一条即填入
   - **手动填数字 SPU ID**（已知 id 时）→ 点"刷新"拉 SKU 列表选一个
   - ~~粘贴商品 URL~~（万代商品页禁止分享，此路不通）
3. **③ 收货**：点"先预检"按钮（见下文）后地址下拉会自动加载
4. **④ 开抢时间**：选日期时间（本地时区 Asia/Shanghai）
5. **⑤⑥**（可选）高级策略 / 推送通知折叠展开调整

**再点"先预检"**：验证 CK 有效 + 拉商品信息 + 拉地址列表。看到绿色 "✅ 预检通过" 就 OK。

**到时间或想开始**：点"开始抢购"红色按钮。切到倒计时视图。

### 倒计时视图你会看到什么

- 巨大的 **T-HH:MM:SS** 夏亚红大字
- 进度条从 0% 走到 100%
- 实时日志流（黑底彩字，最近 300 行）
- 中途想停点"中止"

### 抢到成功

金黄色 **"抢 到 了"** 三个大字出现。下面显示订单 id + prepay_id（可一键复制）。

点 **"打开微信"** 按钮（Windows 下）会通过 `weixin://` 协议唤起微信，你手动进"我的订单"付款。

**⚠️ 15 分钟内必须付款**，否则订单自动取消。

### 抢购失败怎么看

失败视图会给一张**诊断卡片**，按错误类型给出图标 + 提示，例如：
- 🔑 **"CK 已失效"**（黄色左条）→ 让朋友重抓 CK
- 📦 **"库存不足"**（红色左条）→ 抢晚了
- ⏰ **"尚未开售"**（黄色左条）→ 服务端还没放票
- 🛡️ **"被万代风控拦截"**（红色左条）→ 等 5-10 分钟

点"查看原始错误"展开底层具体信息。

### 持久化

GUI 会在以下位置存字段（下次打开预填）：
- Windows：`%APPDATA%/BandaiSniper/state.json`
- Linux：`~/.config/BandaiSniper/state.json`

**不存 CK**（除非勾选"记住 CK"），不存密码，不联网。
