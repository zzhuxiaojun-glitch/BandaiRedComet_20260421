## 15.1 项目定位

一句话：**万代南梦宫上海微商城（PB 限定商品）自动抢购工具**，通过客户端加密 API 直连，1 秒内完成「预检 → 下单 → 拉支付参数」。

面向两类用户：
- **你（你）**：命令行开发 / 调试 / 维护
- **朋友**：图形界面双击即用（正式 exe Phase 3 出）

## 15.2 技术栈速读

| 层 | 选型 | 为什么 |
|---|---|---|
| 核心 HTTP | `httpx` + HTTP/2 + keep-alive | 抢购时 TLS 握手延迟低 |
| 加密 | `pycryptodome`（AES-192-ECB + PKCS7）+ 手搓 md5 | 复现小程序 JS 里老式 md5（UTF-16 截断 bug） |
| 异步 | `asyncio` | 并发 N workers + 库存轮询 + 倒计时 |
| 配置 | `pydantic` + `pyyaml` | 字段校验强 |
| CLI | `click` | 简单清爽 |
| 时间 | `ntplib` + 服务端 `/api/common/config/get` 双校准 | 精度需求 < 100ms |
| 日志 | `loguru` | 彩色 + jsonl 双路落盘 |
| GUI | `pywebview` + 原生 JS（无前端框架） | 零构建、打包小 |
| 打包 | `PyInstaller`（Phase 3 做） | 单 exe |
| 测试 | `pytest` | 31/31 全绿 |

## 15.4 功能模块一览

### A. 加密 / 签名核心（`src/bandai_sniper/crypto.py`）

- `aes_encrypt` / `aes_decrypt` —— AES-192-ECB + PKCS7
- `_sort_and_coerce` —— 还原 JS `o(e, r, t)` 的三种形态（sign vs encrypt，顶层 Array vs Object）
- `_js_str` / `_js_json_dumps_raw` —— 还原 JS `String(142.0) === "142"` 等行为
- `build_signed_request` —— 一站式生成 headers + body + url_suffix，md5 用 `ord(c) & 0xFF` 还原 UTF-16 截断

### B. API 封装（`api.py`）

| 接口 | 用途 |
|---|---|
| `sync_timestamp` | 拉服务器时间，校准 offset |
| `whoami` | 验证 CK + 拿 memberId |
| `get_spu_detail` | 商品详情（库存/价/状态） |
| `get_sku_list` | SKU 列表 |
| `list_addresses` / `get_default_address` | 收货地址 |
| `query_available_coupons` | 可用优惠券 |
| `confirm_order` | 算价（不占库存） |
| `create_order` | 真占库存 + 拉微信支付参数 |
| `get_order_detail` | 查订单状态 |
| `check_agreement` / `is_collect` / `get_near_store` / 等 | 辅助 |

### C. 主循环编排（`sniper.py`）

**流程**：`precheck` → `countdown` → `fire`

- **precheck** 4 步：服务端对时 → whoami → spu_detail → 地址解析
- **countdown** 两段睡：先 NTP 睡到预热窗口 → `sync_timestamp` 再精确校准 → 最后 `wait_until` 忙等到点
- **预热期库存轮询**（瞬爆款优化）：每 150ms 拉 spu_detail，一看到 `stock↑` / `saleStatus→0` / `sellOutFlag→false` + 时间到最早 fire 窗口 → 立刻 `trigger.set()` 提前 fire
- **fire**：N 并发 workers → `confirm_order` → `create_order` → 首个成功者出订单

### D. CLI（`python -m bandai_sniper`）

```bash
python -m bandai_sniper ntp       # 测本机与 NTP 时间偏差
python -m bandai_sniper check     # 不抢，只预检（CK / 商品 / 地址）
python -m bandai_sniper run       # 按 config.yaml 跑完整抢购
python -m bandai_sniper gui       # 启动 GUI（Phase 1+2）
```

### E. GUI（`ui/`）✨

**窗口**：PyWebView + 原生 JS 单页，4 视图切换

| 视图 | 内容 |
|---|---|
| 表单 | 6 个卡片（账号/商品/收货/时间/高级/通知），预检按钮 |
| 抢购中 | 巨大倒计时 + 进度条 + 实时日志 |
| 抢中成功 | 金黄色 "抢 到 了" + 订单号 + prepay_id + 复制按钮 + 打开微信 |
| 失败诊断 | 错误分类卡片（icon + 标题 + 提示）+ 原始错误折叠 |

**配色**：**夏亚扎古 + 阿姆罗高达** 混搭
- 主 `#C8122C`（夏亚红）
- 辅 `#E86A8C`（夏亚粉）/ `#F4F4F0`（高达白）/ `#113D6E`（高达蓝）/ `#F7D200`（高达黄）

**已实现的交互**：
- ✅ 粘贴商品 URL 自动识别 SPU ID（Phase 2.1）
- ✅ 错误分类卡片（14 类：CK失效 / 风控 / 限购 / 库存 / 未开售 / 价格护栏 / 432 / 超时 / 配置错 / 会话冲突 / 解密失败 / worker 全败 / 手动中止 / 兜底）（Phase 2.2）
- ✅ 账号配置持久化（`%APPDATA%/BandaiSniper/state.json`），CK 默认不存
- ✅ 日语原名并排显示（商品预览区）
- ✅ SKU 下拉自动加载
- ✅ 试发通知 按钮（bark / server_chan / feishu）
- ✅ 订单号 / prepay_id 一键复制
- ✅ "打开微信"快捷（Windows 协议 `weixin://`）

### F. 通知推送（`notify.py`）

支持三通道：**bark** / **server_chan** / **feishu**。抢到或失败都会推送。

### G. 日志系统

- `logs/run_TIMESTAMP.log` —— 完整 loguru 日志（彩色）
- `logs/requests_TIMESTAMP.jsonl` —— 每个 HTTP 请求的结构化记录
- `logs/gui_TIMESTAMP.log` —— GUI 模式专用

## 15.8 非功能亮点

- **真实订单验证**：不只是离线算法验证，真打通过服务端
- **中文踩坑全记**：17 条陷阱详细记录，包括 JS md5 UTF-16 bug 这种极其隐蔽的坑
- **测试驱动**：31 个单元测试锁住算法边界，以后升级依赖不会悄悄坏
- **双端开发**：WSL 开发 + Windows 打包，朋友最终拿到的是单 exe
- **无依赖前端**：没有 webpack / react / vite，单 HTML+CSS+JS，维护成本低
