# 3. 抓包指南（v2 · 自动化版）

> 目标读者：**运维者本人**（不是朋友）
> 目的：抓一次完整的人工下单流量，交付 HAR 给后续实现 `api.py` 使用
> 相关文档：[1_CK分析与方案建议](./1_CK分析与方案建议.md) · [2_PRD](./2_PRD.md)

---

## TL;DR · 只需 5 步

所有脚本都在 [`tools/capture/`](./tools/capture/) 目录，双击就能跑。

1. **打开 Windows** → 登录微信 PC → 打开万代小程序，登录自己的号（**新号**）
2. **管理员 PowerShell 右键运行** `tools/capture/start_capture.ps1`
3. 在小程序里按 §3 的清单点 7 下（大约 2 分钟）
4. 回到 PowerShell 按 `Ctrl+C` 停止，生成 `bandai_capture.har`
5. 跑两条命令做自检和脱敏：
   ```
   python inspect_har.py bandai_capture.har      # 看抓全了没
   python sanitize_har.py bandai_capture.har     # 脱敏
   ```
6. 把 `bandai_capture.sanitized.har` 丢到 `captures/` 目录 → 完工

## 1. 为什么不用 Proxifier 了

旧版方案需要装 Proxifier + 配规则 + 装根证书 + 可能还要破 SSL Pinning。

**新方案**用 `mitmproxy --mode local:WeChat,WeChatAppEx` —— 这是 mitmproxy 10+ 的"按进程名透明劫持"模式，底层用 WinDivert 抓 TCP 包，**完全不需要装证书、不需要 Proxifier、不需要配代理**。

代价：需要管理员权限（驱动需要）。但 `start_capture.ps1` 会自动弹 UAC。

## 3. 点击清单（抓包时按顺序操作）

**开抓之前**：

- 运行 `start_capture.ps1`，看到 `mitmdump` 开始吐日志就是 OK
- 保持 PowerShell 窗口在前台可见

**在小程序里按顺序点**（慢一点，2 秒一步）：

| # | 动作 | 为什么 |
|---|---|---|
| 1 | 刷新 / 重进小程序 | 触发登录态相关接口 |
| 2 | 首页随便滑一下 | 拿到 /home 类接口（非必需但有用）|
| 3 | 搜一个**便宜的现货商品**（20 元级周边） | 拿到 /search 接口 |
| 4 | 进商品详情页 | **必抓** /product/detail 或 /goods/detail |
| 5 | 点"立即购买"或"加入购物车" | /cart/add 或 /order/prepare |
| 6 | 确认订单页：确保默认地址正确，数量 1 | /order/confirm |
| 7 | **点"提交订单"** | **必抓** /order/create → 拿到订单号 |
| 8 | 支付页弹出后 **不要付款**，直接关掉 | —— |

> ⚠️ 第 7 步会真的创建一笔未支付订单，几分钟后自动取消。放心操作。

**停止抓包**：PowerShell 窗口按 `Ctrl+C`，HAR 自动保存。

## 4. 自检与脱敏

```powershell
# 在 tools/capture/ 目录下
cd $PSScriptRoot\tools\capture

# 看抓全了哪些接口（不改文件）
python inspect_har.py bandai_capture.har

# 脱敏（会新生成 .sanitized.har）
python sanitize_har.py bandai_capture.har
```

### inspect_har.py 输出样例

```
📊 bandai_capture.har · 共 42 条请求

── 域名分布 ──
    42  rm-app-api.bandainamcoshanghai.com

── URL 路径（去参数） ──
      3× GET   /api/home/banner
      1× GET   /api/user/profile
      2× GET   /api/product/detail
      1× POST  /api/cart/add
      1× POST  /api/order/confirm
      1× POST  /api/order/create
      ...

── 预期接口检查 ──
  ✅ 用户信息 / CK 校验     命中 ['/api/user/profile']
  ✅ 商品详情              命中 ['/api/product/detail']
  ✅ 加购物车              命中 ['/api/cart/add']
  ✅ 确认订单              命中 ['/api/order/confirm']
  ✅ 创建订单              命中 ['/api/order/create']
  ...

🎉 看起来抓得挺全。下一步跑 sanitize_har.py 脱敏。
```

如果有 ❌：**说明那一步没点到**，重新跑一次 `start_capture.ps1` 补抓即可（mitmdump 启动时会覆盖 HAR，所以一次跑完）。

### sanitize_har.py 脱敏项

自动替换以下字段：

- `api-access-token` header 值 → `REDACTED_API_ACCESS_TOKEN_len<N>`（保留长度用于排查）
- 手机号 `1[3-9]\d{9}` → `REDACTED_PHONE`
- 身份证 → `REDACTED_IDCARD`
- `openid / unionid` → `REDACTED_OPENID`
- 微信支付 `prepay_id / paySign / nonceStr` → `REDACTED_*`
- URL query string 里的 `token / access_token / openid`

## 6. 常见坑 & 应对

| 现象 | 原因 | 解决 |
|---|---|---|
| `start_capture.ps1` 提示"执行策略" | PowerShell 默认不让跑本地脚本 | `Set-ExecutionPolicy -Scope CurrentUser RemoteSigned` 一次就好 |
| 弹窗"Windows Defender 已阻止" | WinDivert 驱动误报 | 点"允许"即可，这就是 mitmproxy 装的合法驱动 |
| mitmdump 启动但抓不到流量 | 微信进程名不一致（企业版/官方版） | 任务管理器看实际进程名，改 `start_capture.ps1` 里的 `-Processes 参数` |
| HAR 巨大（>50MB） | 某些接口有长轮询 | bandai_filter.py 已经按域名过滤，如果还大就是万代接口自己在刷 |
| 小程序报"网络异常" | 极少数接口有 cert pinning | 跳过它，只要下单链路的 5 个接口抓到就行 |
