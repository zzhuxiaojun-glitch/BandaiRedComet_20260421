### 今天踩的坑 & 修复

**坑 1：AES key 看起来像 base64 但其实不是**
- `secretKey = "5p/9OgW9F5cJc0fOs0TXYA=="` 看着像 base64（末尾 `==`），其实 JS 里用的是 `CryptoJS.enc.Utf8.parse(secretKey)`，**把这个字符串按 UTF-8 字面量当 24 字节 key**。AES-192。
- 一开始我差点 `b64decode` 成 16 字节 key 搞错，还好对着 JS 源码确认。

**坑 2：签名算法对嵌套结构的处理**
- 我第一版把 JS `o(e, r, t)` 里的 `t` 误读成 sign/encrypt 开关，结果嵌套数组全被我包成 `{"str": "..."}`，12 个样本只对 1 个。
- 实际：`t` 是 `encryptionEnable`（线上恒为 `true`）。**顶层 Array 才包 `{"str": JSON.stringify(arr)}`（仅 sign 路径），嵌套的 Object / Array 在 sign 和 encrypt 两条路径都保持原样**，只有叶子 primitive 走 `String(v)`、`null` 丢弃。
- 修正后：12/12 全通过。
