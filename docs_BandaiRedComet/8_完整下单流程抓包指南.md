# 8. 完整下单流程抓包指南

**目的**：补抓上次 HAR 里缺的下单半段（商品详情 → 地址 → 确认订单 → 提交订单 → 拉支付参数）。
**前置**：[3_抓包指南.md](./3_抓包指南.md) 已跑通过一次，环境可复用。
**输出**：一份含完整下单链路的 `bandai_capture.har`（含明文 ck，**本地保管不 commit**）+ 一份 `bandai_capture.sanitized.har`（脱敏后可给我看）。

---

## 8.1 为什么要再抓一次

第一次 HAR（共 42 条请求）**只覆盖了"浏览流"**：首页 / 分类 / 商品列表 / 推荐。
缺的是**下单流**，下面这些接口**一次都没出现**：

| 缺失接口 | 典型路径 | 为什么必须抓 |
|---|---|---|
| 商品详情 | `GET /commodity/v1/app/spu/v2/detail/new` | 拿 `skuId` 必需（spuId≠skuId） |
| 地址列表 | `GET /member/v1/app/address/list`（猜的） | 下单要 addressId |
| 确认订单 | `POST /order/v1/app/order/confirmOrder` | 试算价格、确认库存 |
| 创建订单 | `POST /order/v1/app/order/createOrder` | **真占库存**，响应里带微信支付参数 |
| 拉支付参数 | 可能合并在 createOrder，也可能单独接口 | 最后一步唤起微信支付 |

没有这些接口的真实请求/响应，`api.py` 只能靠猜字段名写，临抢购才调试太晚。

---

## 8.2 抓前准备

### 选一个"安全商品"

这次要走到"提交订单"（真生成订单占库存），所以商品**必须同时满足**：

- ✅ **常规在售**，**不是** PB 限定 / 抽选 / 倒计时抢购
- ✅ **单价低**（50 元以下最好，方便心理上不怕）
- ✅ **库存充足**（商品详情页应该直接显示"加入购物车 / 立即购买"，不是"即将售罄"）
- ✅ **不免邮**（能走到完整运费计算最好，免邮商品可能跳过某些接口）

推荐方向：
- 小型手办 / 模型周边 / 拼装模型的补品配件 / 积分俱乐部周边等

**⚠️ 千万避免**：抢购商品、PB 限定、抽选品 —— 一旦走到"提交订单"真抢到，得退款 + 占库存 + 触发风控。

### 检查环境

按 [3_抓包指南.md](./3_抓包指南.md) 的方法，确认：

- 管理员 PowerShell 可用
- `tools/capture/start_capture.ps1` 之前跑过一次不报错
- 微信 PC 登录的是你自己的号（**不是朋友的**）
- 万代小程序里有至少一个收货地址（没有就先加一个）

## 8.3 抓包点击清单

**开抓前**：
- 管理员 PowerShell 跑 `tools\capture\start_capture.ps1`
- 看到 mitmdump 开始吐日志（有 `bandainamcoshanghai.com` 的行在滚）
- **输出文件名改一下**：在 `start_capture.ps1` 里把 `$OutFile` 改成 `bandai_capture_order.har`，别覆盖上次抓的 browse HAR

**然后在小程序里按顺序点**（慢一点，每步等 2 秒让接口都回）：

| # | 动作 | 会触发的接口（预期） |
|---|---|---|
| 1 | 从小程序首页进入（已登录） | 首页一堆浏览接口，**会重复 browse HAR**，没关系 |
| 2 | 搜或点进一个**符合 8.2 条件的安全商品** | `spu/v2/detail/new` ⭐ |
| 3 | 商品详情页往下滑一次 | 可能触发评价/推荐接口 |
| 4 | 点"选规格"或"立即购买"，选一个 SKU | 可能有 `sku/detail` |
| 5 | **点"立即购买"**（不走购物车） | 进入确认订单页，`order/confirmOrder` ⭐ |
| 6 | 确认订单页停留 2 秒 —— 看清地址、SKU、价格 | 可能有 `address/list`、`coupon/available` 等 |
| 7 | 点"切换地址"并选一次（哪怕选回原来的） | **一定会触发** 地址接口 ⭐ |
| 8 | 回到确认订单页，**数量改成 2，再改回 1** | 触发一次 re-confirm，对照能看出 quantity 字段 |
| 9 | **点"提交订单"** ⭐⭐⭐ | `order/createOrder` + 微信支付参数接口 |
| 10 | 微信支付弹窗出现 → **直接关闭，不要付** | 到此为止 |
| 11 | 回到小程序，**进"我的订单"看到刚才那笔未支付** | 确认订单真的生成了（后续万代会自动取消） |

**停止抓包**：PowerShell 按 `Ctrl+C`，等"saved to ..."提示出现再关窗口。

## 8.4 抓后自检

```powershell
cd <工作区>\tools\capture
python inspect_har.py bandai_capture_order.har
```

**必须看到这几行 ✅**（路径可能略不同，关键词对就行）：

```
✅ 商品详情 (spu/detail)     命中 ['/commodity/v1/app/spu/v2/detail/new']
✅ 确认订单 (confirmOrder)   命中 ['/order/v1/app/order/confirmOrder']
✅ 创建订单 (createOrder)    命中 ['/order/v1/app/order/createOrder']
✅ 地址列表                 命中 ['/member/.../address/...']
```

如果 `createOrder` ❌ 或 `confirmOrder` ❌：**重抓**，最常见原因是第 9 步点了"提交订单"但弹窗先出来了就关了导致网络请求被掐断。慢一点点，等"订单生成中..."转圈完才关弹窗。

## 8.5 脱敏 + 交付

```powershell
python sanitize_har.py bandai_capture_order.har
# 产出 bandai_capture_order.sanitized.har
```

脱敏规则自动处理：`api-access-token` / 手机号 / 身份证 / openid / 微信支付 `prepay_id`/`paySign`/`nonceStr` → 全部 mask。

然后把两份文件都放到工作区的 Windows 目录（本地保管，不 commit）：

```

  bandai_capture_order.har              # 明文，自己看用
  bandai_capture_order.sanitized.har    # 脱敏，可给我
```

## 8.6 注意与兜底

**触发了风控/报错怎么办**：
- 如果小程序在第 9 步提示"请求过于频繁"或"请稍后再试"，说明服务端已经在限流，退出小程序等 10 分钟再抓
- 如果 `inspect_har.py` 显示同一接口出现了 4+ 次，可能是某个接口在长轮询，不影响后续分析

**抓到了订单但想取消**：
- 进万代小程序 → "我的订单" → 找到那笔未支付 → 点"取消订单"即可
- 或者就不管它，未支付订单一般 15 分钟自动释放库存

**整个过程预计耗时**：5 分钟（3 分钟点击 + 2 分钟自检脱敏）。
